// why it's different

Judgment & safety, not just another scanner

What you get over running the same tools by hand.

▹Report-derived methodology

Each hunt-* skill carries a report_count and sources: list (hunt-idor: 26 reports, hunt-sqli: 12) and anchors to real current CVEs — Rocket.Chat CVE-2021-22911, Mongoose CVE-2024-53900, Django CVE-2024-42005, vCenter CVE-2021-21972 → 2024-37085. Not generic checklists.

▹False-positive discipline built in

Every quality skill leads with a kill gate. The global 7-Question Gate + 4-gate validator + never-submit list protect submission-validity ratio — the metric manual hunters most often torch.

▹Crown-jewel targeting

The 5-minute and 20-minute rotation rules steer effort to where each vuln class actually pays, and abandon dead ends fast. Depth over breadth, follow the money.

▹Deterministic scope safety

Code-not-LLM scope checking on every outbound request: anchored-suffix matching, fail-closed, IP/CIDR refused — a guarantee ad-hoc tooling cannot offer.

▹Cross-target learning

A technique that paid off on one React/Next.js target surfaces automatically on the next (partial tech-stack overlap matching), so the operator gets cumulatively better.

▹Autonomous closed loop

Autopilot runs scope → recon → rank → hunt → validate → report with circuit breakers, rate limits, safe-method gating, and a hard human-approval boundary before submission and before any live credential attack.

// architecture

An eight-layer stack

A request flows top-to-bottom; each layer has a single responsibility.

1
Knowledge skills/ · declarative
How to think / what to check. Pure methodology, no side effects. Auto-loaded by topic; bb-methodology is the master router.
2
Interface commands/
42 slash-command entry points. Thin routers — invoke an engine script directly or dispatch to an agent/skill. Prefixed to avoid collisions (/pickup not /resume).
3
Actor agents/
Stateful subagents that run multi-step loops, pick tools, and carry safety rails in their own prompts. Pinned models (Haiku/Sonnet/Opus).
4
Engine tools/ scripts/ engine/ · deterministic
Side-effecting execution. tools/hunt.py orchestrator; engine/skill_map.py maps surface → class → skill → first curl.
5
Integration mcp/ · 6 servers
Proxy bridges (Burp/Caido) + read-only platform intel (HackerOne, Bugcrowd, Intigriti, Immunefi).
6
State memory/ · dashboard/
Persistence: hunt-memory JSONL + SQLite findings DB + read-mostly Flask web GUI at 127.0.0.1:5000.
7
Safety scope_checker · audit_log · rules/
Deterministic scope gate, accountability audit trail, always-active behavioral rules.
8
Quality lint_skills · CI · tests/ eval/
Pre-merge lint + 372-test CI + accuracy eval harness.

// end-to-end workflow

The hunting lifecycle

A 5-phase non-linear methodology mapped to commands — from scope to human-approved submission.

SCOPE /scope

Define/Select/Execute discipline; pick identity (anon vs authenticated); verify each asset in scope; choose Wide vs Deep route.

RECON /recon → recon_engine.sh

subdomain enum → httpx + tech detect → nmap → gau/wayback/katana → JS secret grep → ffuf → CI/CD scan → nuclei → takeover leads. 5-min kill on all-403 / static-only / 0 nuclei.

MAP & RANK /surface · recon-ranker

Understand the app like its developer; classify and rank surface (api/auth/admin/idor-candidates); prioritize where broken auth hurts most; favor less-saturated classes.

HUNT /hunt → hunt.py → vuln_scanner.sh

Scripted scan first, then manual deep-dive by class; error → time → OOB → boolean; A→B sibling hunting; 20-min rotation with hard stops.

PROVE & ESCALATE

Turn Low into Critical — XSS→ATO, IDOR→PII scrape/ATO, SSRF→metadata→RCE, SQLi→shell; minimize prerequisites; quantify $ impact.

VALIDATE /validate

7-Question Gate (one NO = kill) + cross-identity proof + 4 gates (sanity → impact → dedup → report quality). Output: PASS / KILL / DOWNGRADE.

REPORT /report

Only after PASS: fixed title formula, impact-first, no "could potentially," working PoC, CVSS 3.1, < 600 words, platform-specific.

SUBMIT (human-approved) + post-submission

Never auto-submitted; final pre-submit checklist; re-test deployed fixes for incomplete-patch bypass; log outcome with /remember.

Autopilot modes /autopilot

Same scripts and outputs as the manual chain, driven by one agent loop. Scope-check every URL, log every request, never auto-submit, circuit breaker after 5 consecutive 403/429/timeout.

--paranoid
Default. Stops on every finding / signal.
--normal
Stops after each validation batch.
--yolo
Runs until surface exhausted; still needs report + PUT/DELETE/PATCH approval.
--quick
~40% fewer tokens.

// safety & memory model

Safe on every request, cumulative across targets

Four memory/ modules plus the deterministic scope checker. Runtime JSONL lives outside the repo at ~/.claude/projects/{project}/hunt-memory/.

⊘Scope checker

Deterministic, code-not-LLM. Anchored-suffix matching (*.target.com accepts sub.target.com, rejects evil-target.com), exclusions before allowlist, fail-closed, IP/CIDR refused. Called before any outbound request.

⊘Audit log

Append-only JSONL of every outbound request with flock + partial-write detection. Records only a non-secret 12-char session hash — raw cookies/tokens are never written.

⊘Circuit breaker + rate limit

Per-host circuit breaker (threshold 5, cooldown 60s). Two-tier rate limiter (recon 10/s, test 1/s). SafeMethodPolicy: GET/HEAD/OPTIONS safe, all others require approval.

⊘Pattern learning

pattern_db.py: one JSON object per line keyed by vuln class + tech stack. match() is exact on vuln_class, partial-overlap on stack — so a React IDOR technique surfaces on the next React target. Sorted by payout then recency.

⊘Credential gate

--with-credential-attack required for wordlist-gen / osint-employees / spray (TREVOR o365/okta). credential-hunter agent hard-stops before live /spray; spray adds typed-hostname confirm + lockout warning.

⊘Rotation + schema

rotation.py: 10 MB cap, 3 backups, multi-process safe. schemas.py: schema v1, rejects unknown fields, closed enums, ISO-8601 timestamps; auto-logs an end-of-session summary so memory populates without manual /remember.

Division of responsibility: scope_checker decides whether a host may be touched at all; AutopilotGuard decides whether this specific request is allowed right now; AuditLog records verifiable evidence; rotation keeps it bounded; schema validation keeps it well-formed.

// quality & CI

Pre-merge lint + 372 tests

A single workflow .github/workflows/skill-lint.yml with a dorny/paths-filter gate driving two jobs.

▸Job 1 — lint

Stdlib-only lint_skills.py (Python 3.12, exits 0/1). Checks frontmatter integrity (name == dir, description ≤ 1024, body ≤ 500 lines), leaked/duplicate keys, fence balance, a SHA-256 identifier denylist, and a real-secret scan (AWS AKIA, private keys, JWTs, Slack/Google/GitHub tokens) with an example allowlist.

▸Job 2 — tests

372 test functions across 23 pytest files + 1 shell test + shared conftest.py. Largest: test_auth_session (45), test_token_scanner (39), test_recon_adapter (31), test_scope_checker (30), test_schemas (23), test_audit_log (22), test_rotation (22). An eval/ harness (PortSwigger labs, false-positive cases) measures real hunting accuracy out-of-band, including eval/code_corpus/ — a white-box ground-truth corpus (vulnerable + safe variants) scored by run_eval_code.py for precision/recall/F1.
