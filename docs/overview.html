// the thesis

Judgment & safety, not just another scanner

Scanners produce findings; this plugin produces decisions. Deterministic engines (Semgrep, osv-scanner, scope checker, PoC replayer) do the finding and the math; the model reasons over the results — confirming reachability, killing false positives, and gating every outbound request through code, not vibes.

◆Engine finds, model triages

Real SAST/SCA emit normalized findings with stable fingerprints; the model confirms exploitability and ranks impact. Scored at F1 1.00 on the committed white-box corpus.

◆Closed-loop verification

Findings persist with a replayable poc_spec. /retest replays them and returns FIXED / STILL-VULN / REGRESSED — proving a fix held instead of re-running the whole hunt.

◆Fail-closed by default

Scope is decided in code. Unmatched hosts return NEEDS_REVIEW (not a silent reject). Every request passes the scope checker, audit log, rate limiter and circuit breaker first.

// new in this release

What just landed

Real static + composition analysis, a measured white-box benchmark, findings that persist and replay, scope confirmation, diff-scoped PR review, source-map recovery, real DOM-XSS proof, secret validation, and a GUI workbench.

Real SAST + SCA F1 1.00

/sast drives Semgrep (regex fallback) into normalized, fingerprinted findings; /sca reads lockfiles via osv-scanner / pip-audit into severity-ranked advisories. Scored by the eval/code_corpus/ ground-truth benchmark.

semgrep → normalize → fingerprint → triage

Findings persistence + replay

save_finding.py stores a verified finding with a machine-replayable poc_spec. /retest --from-db and the GUI re-run it to a FIXED / STILL-VULN / REGRESSED verdict.

save_finding → DB(poc_spec) → /retest → verdict

Diff-aware PR review

/pr-review + the diff-aware-pr-reviewer agent score ONLY a PR's changed lines, splitting findings into NEW vs pre-existing and posting inline comments. Read-only; never pushes.

git diff → SAST(changed) → NEW vs legacy → inline comments

Source-map analyzer

/js-analyze parses //# sourceMappingURL, extracts sourcesContent back to readable source, then runs SAST + a secret pass. Beautifies the bundle when there is no map.

bundle → .map → sourcesContent → SAST + secrets

DOM-XSS verifier

/dom-verify fires one (url, payload) in headless Chromium (Playwright). A dialog, window[marker] or console marker flips a [POSSIBLE] sink to CONFIRMED — or kills it.

[POSSIBLE] → headless Chromium → CONFIRMED / killed

Secret validation + org regexes

secret_validate.py reconciles trufflehog / gitleaks / noseyparker hits, runs liveness checks, and matches org-specific token formats via custom_secret_patterns.py before DB ingest.

3 scanners → reconcile → liveness → custom regex → DB

Scope NEEDS_REVIEW

Unmatched hosts, bare IPs and sibling/acquisition hosts now return NEEDS_REVIEW instead of a silent reject. Interactive --confirm mode requires an explicit operator assertion to proceed.

host → in / excluded / NEEDS_REVIEW → operator confirm

GUI workbench

The dashboard/ Flask app adds auth, finding/report detail views, Chart.js charts, export, and one-click re-run / verdict over persisted findings — on top of the live recon/hunt TUI.

findings DB → web UI → re-run · verdict · export

White-box code benchmark

eval/code_corpus/ holds 14 vulnerable + 14 safe variants with a committed findings fixture, scored to precision / recall / F1 per class — making code-audit accuracy a measured number, not a claim.

vuln+safe corpus → score() → P / R / F1

// safety & memory model

Safe on every request, cumulative across targets

Four memory/ modules plus the deterministic scope checker. Runtime JSONL lives outside the repo at ~/.claude/projects/{project}/hunt-memory/.

⊘Scope checker

Deterministic, code-not-LLM. Anchored-suffix matching (*.target.com accepts sub.target.com, rejects evil-target.com), exclusions before allowlist, fail-closed, IP/CIDR refused. Returns NEEDS_REVIEW for unmatched/sibling hosts. Called before any outbound request.

⊘Audit log

Append-only JSONL of every outbound request with flock + partial-write detection. Records only a non-secret 12-char session hash — raw cookies/tokens are never written.

⊘Circuit breaker + rate limit

Per-host circuit breaker (threshold 5, cooldown 60s). Two-tier rate limiter (recon 10/s, test 1/s). SafeMethodPolicy: GET/HEAD/OPTIONS safe, all others require approval.

⊘Pattern learning

pattern_db.py: one JSON object per line keyed by vuln class + tech stack. match() is exact on vuln_class, partial-overlap on stack — so a React IDOR technique surfaces on the next React target. Sorted by payout then recency.

⊘Credential gate

--with-credential-attack required for wordlist-gen / osint-employees / spray (TREVOR o365/okta). credential-hunter agent hard-stops before live /spray; spray adds typed-hostname confirm + lockout warning.

⊘Rotation + schema

rotation.py: 10 MB cap, 3 backups, multi-process safe. schemas.py: schema v1, rejects unknown fields, closed enums, ISO-8601 timestamps; auto-logs an end-of-session summary so memory populates without manual /remember.

Division of responsibility: scope_checker decides whether a host may be touched at all; AutopilotGuard decides whether this specific request is allowed right now; AuditLog records verifiable evidence; rotation keeps it bounded; schema validation keeps it well-formed.

// quality & CI

Pre-merge lint + 917 tests

A single workflow .github/workflows/skill-lint.yml with a dorny/paths-filter gate driving two jobs.

▸Job 1 — lint

Stdlib-only lint_skills.py (Python 3.12, exits 0/1). Checks frontmatter integrity (name == dir, description ≤ 1024, body ≤ 500 lines), leaked/duplicate keys, fence balance, a SHA-256 identifier denylist, and a real-secret scan (AWS AKIA, private keys, JWTs, Slack/Google/GitHub tokens) with an example allowlist.

▸Job 2 — tests

917 test functions across 44 test files + shell tests + shared conftest.py. New coverage includes test_sast_tool, test_sca_tool / test_sca_audit, test_pr_diff_review, test_retest, test_findings_persistence, test_dom_xss_verifier, test_sourcemap_analyzer, test_secret_validate, test_code_corpus_eval / test_run_eval_code, and test_dashboard_gui. The eval/code_corpus/ white-box benchmark scores precision / recall / F1 deterministically — no scanner needed in CI.
